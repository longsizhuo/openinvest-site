# 18 — 治理章程：口径定义 / 判据 / 脏数据红线 / 否决权（用户签字）

> 这份是**治理章程**，不是技术文档。AI 起草、用户签字。用户签的不是"算法对不对"
> （那是 AI + 确定性回算 + transcript 校验的事），是 **"标准是不是事先定的、有没有在
> 看到结果后被改过、判据冲突时谁裁决"**。任何改生产行为的 PR、任何"丢哪些数据/选哪个
> 判据"的决定，都要落到这里、对照这里。
> 起因：2026-06-13 口径审计——先发现一个根因 bug，**AI 又把它的影响夸大了一个数量级、
> 用错误基线做了"证明"、并单方面切了生产**（已回滚）。治理空隙必须正式关闭，且关闭方式
> 本身要防住"AI 起草判据、AI 自己选、AI 报数让用户信"的递归漏洞。

## 0. 三条核心原则（2026-06-13 加，关闭递归漏洞）

### A. 历史重算只认决策当时的冻结快照，不用今天重拉的数据

任何历史重算/回算，**唯一可信输入是决策当时落盘的快照**——committee transcript 冻结的值、
账本/verdict_review 落盘原值。**绝不用今天重新拉取的数据**，覆盖一切外部输入、**包括价格**
（不只是新闻）。实证：6/9 transcript 冻结 `VIX=21.4`，今天 DB 已被 yfinance 修订成 `19.9`。
- 没当时快照的历史点 → 标"**不可忠实复现，仅近似**"，结论不得依赖其精确值。
- **重算脚本必须先用 ≥3 个有 transcript 的冻结点反向校验**，复现冻结值才可信
  （先例 `scripts/audit_convention_diff.py` v2：4 点全复现 730 窗）。脚本是被验证对象，
  transcript 是金标准，不许反过来"信脚本"。

### B. 用户显式否决权（写进章程本身，不只是"签字同意章程"）

即使预注册判据给出 PASS/FAIL，**用户有权签字推翻该结论**，记录在案即可（写明推翻哪条
判据、理由、日期）。判据 PASS ≠ 自动改生产；判据 FAIL ≠ 自动否决。**判据是输入，不是裁决；
最终决定权在用户。**

### C. 判据：起草 / 人话意图 / 跑前批准 / 冲突裁决

- 判据由 AI 起草，但**起草时必须附一句人话意图解释**——这条判据想捕捉/守护什么，用用户
  看得懂的话写，让用户能在跑之前**有意义地批准**它（而不是批准一串自己看不懂的阈值）。
- 判据**必须在跑实验/回算之前经用户批准**（口头或文字），跑后不许改（认知泄漏红线）。
- **判据冲突由用户裁决**，不许 AI 单方面定；冲突 + 选定结果记入 §5 审计 log。
  实例：黄金防御判据——论文用"中位右偏"、本地脚本用"p_below"，AI 单方面选了后者（空隙）。

## 2. 判据标准（预注册，不许事后改）

- 判据**跑前写死在脚本 docstring + 附人话意图（§0-C）**，跑后不许动。先例：
  `fit_path_calibration.py`、`validate_gold_defense.py`、TA `eval_ta_signal.py`。
- **冻结自由度再回算**：实现者已知结局时"看结果再微调"是认知泄漏，比代码 bug 难防。
- 判据冲突按 §0-C 由用户裁决。

## 3. 脏数据 / 前视红线（最硬）

- **新闻/情绪/基本面：只用决策当时落盘快照**，缺了标"缺失"，**绝不用今天重抓的补**
  （事后修订 + 索引错位 + 已知结局 = 前视污染，无法自证）。
- **纯价格衍生量可合法重算**（只依赖 ≤当天收盘、无前视），**但仍受 §0-A 约束**：历史点
  无冻结快照时只能近似，且重算脚本须经 transcript 校验。
- 缺失即缺失，不补值、不用未来均值回填。

## 4. 合并前置条件（生产行为变更）

任何改生产行为的 PR **必须**附：
1. **一句话信号影响翻译**：「这个改动会让系统在哪些情况下，把原本的 X 信号变成 Y 信号」。
   答不出 = 不知影响面 = 不该合。
2. **确定性回算差异报告 + transcript 校验**（纯价格类）：列出过去哪几天信号会变、为什么，
   人话呈现 + 每日明细可独立复算（先例 `audit_convention_diff.py` v2 + CSV）。

## 6. 红线（这次定，长期守）

- ❌ 不重抓历史新闻/情绪/基本面/**价格**重算（§0-A、§3）。
- ❌ 不在看到回算结果后调任何判据/阈值/窗口（§0-C、§2）。
- ❌ 不靠"观察几天/感觉对不对"给生产签字。
- ❌ 不擅自改生产行为（本次 AI 已违规 → 回滚 → 入 log）。
- ❌ 不急着合积压 PR（#37/#41/#42）：根因改历史序列，下游验收/账本依附旧口径，
  先定口径 → 全部对齐 → 一起合。
- ⚠ regime-lock"方向锁=杠杆旋钮"**只作相对结论**，不作绝对判断。
